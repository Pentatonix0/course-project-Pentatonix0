# Security Policy

## Поддерживаемые версии

| Версия               | Поддержка |
| -------------------- | --------- |
| release (актуальная) | ✅        |
| старые релизы        | ❌        |

## Сообщение о проблемах безопасности

Если вы нашли уязвимость или ошибку, связанную с безопасностью:

1. Не создавайте публичный issue.
2. Отправьте описание на личный email разработчика (или через GitHub Security Advisories).

Пожалуйста, включите:

-   Подробное описание проблемы
-   Минимальный пример, как её воспроизвести
-   Ваши предложения по исправлению

## Практики безопасности

-   Используется **JWT** для аутентификации пользователей
-   Все приватные маршруты требуют **AuthN/AuthZ**
-   Тесты покрывают проверку прав доступа и валидацию данных
-   Пароли хранятся только в виде хэшей (bcrypt)
-   Используется валидация входных данных через **Pydantic**

## Рекомендации для пользователей

-   Не используйте общие токены доступа
-   Регулярно обновляйте зависимости (`pip install -U -r requirements.txt`)
-   Не храните реальные секреты в `.env.example` или публичных репозиториях
